                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>We currently load many plugins with a Map or NamedList – since <a href="https://issues.apache.org/jira/browse/SOLR-215" title="Multiple Solr Cores - remove static singleton" class="issue-link" data-issue-key="SOLR-215"><del>SOLR-215</del></a>, the current core is not available through SolrCore.getSolrCore() and may need to be used for initialization.</p> 
                  <p>Ideally, we could change the init() methods from:</p> 
                  <div class="panel" style="border-width: 1px;">
                   <div class="panelContent"> 
                    <p>void init( final Map&lt;String,String&gt; args );</p> 
                   </div>
                  </div> 
                  <p>to</p> 
                  <div class="panel" style="border-width: 1px;">
                   <div class="panelContent"> 
                    <p>void init( final SolrCore core, final Map&lt;String,String&gt; args );</p> 
                   </div>
                  </div> 
                  <p>Without breaking existing APIs, this change is difficult (some ugly options exist). This patch offers a solution to keep existing 1.2 APIs, and allow access to the SolrConfig and SolrCore though ThreadLocal. This should be removed in a future release.</p> 
                  <div class="panel" style="border-width: 1px;">
                   <div class="panelContent"> 
                    <p> DeprecatedPluginUtils.getCurrentCore();<br> DeprecatedPluginUtils.getCurrentConfig();</p> 
                   </div>
                  </div> 
                  <p>This patch removes the SolrConfig.Initalizable that was introduced in <a href="https://issues.apache.org/jira/browse/SOLR-215" title="Multiple Solr Cores - remove static singleton" class="issue-link" data-issue-key="SOLR-215"><del>SOLR-215</del></a>.</p> 
                  <p>For background, see:<br> <a href="http://www.nabble.com/Initializing---break-init%28%29-API-compatibility--tf4808463.html" class="external-link" rel="nofollow">http://www.nabble.com/Initializing---break-init%28%29-API-compatibility--tf4808463.html</a></p>
