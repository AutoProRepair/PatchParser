                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>1. The syntax boxes in many of the Derby manuals seem to output with extra End-of-line feeds in the PDF's. Some syntax boxes print one word per line. For example:</p> 
                  <p><a href="http://incubator.apache.org/derby/docs/tools/tools-single.html#rtoolsijpropref10135" class="external-link" rel="nofollow">http://incubator.apache.org/derby/docs/tools/tools-single.html#rtoolsijpropref10135</a></p> 
                  <p>This might be a bug with the DITA toolkit, because the DITA source files dont have these End-of-line feeds in them.</p> 
                  <p>This bug was originally reported in the doc reviews for version 10.1:<br> <a href="http://issues.apache.org/jira/browse/DERBY-383" class="external-link" rel="nofollow">http://issues.apache.org/jira/browse/DERBY-383</a><br> (see Myrna's comments)</p> 
                  <p>2. Based on <a href="http://issues.apache.org/jira/browse/DERBY-384" class="external-link" rel="nofollow">http://issues.apache.org/jira/browse/DERBY-384</a> comments to the doc review (see Sunitha's comments), we need to figure out how to et the table numbers to ascend. Currently, they all output as table 1.</p> 
                 </div> 
                </div>
