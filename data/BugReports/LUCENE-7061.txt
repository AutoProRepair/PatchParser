                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>There are still some major problems today:</p> 
                  <p>XYZPoint.newRangeQuery has "brain damage" from variable length terms. nulls for open ranges make no sense: these are fixed-width types and instead you can use things like Integer.maxValue. Removing the nulls is safe, as we can just switch to primitive types instead of boxed types.</p> 
                  <p>XYZPoint.newRangeQuery requires boolean arrays for inclusive/exclusive, but thats just more brain damage. If you want to exclude an integer, you just subtract 1 from it and other simple stuff.</p> 
                  <p>For the apis, this means Instead of:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java"><span class="code-keyword">public</span> <span class="code-keyword">static</span> Query newRangeQuery(<span class="code-object">String</span> field, <span class="code-object">Long</span> lowerValue, <span class="code-object">boolean</span> lowerInclusive, <span class="code-object">Long</span> upperValue, <span class="code-object">boolean</span> upperInclusive);
  
<span class="code-keyword">public</span> <span class="code-keyword">static</span> Query newMultiRangeQuery(<span class="code-object">String</span> field, <span class="code-object">Long</span>[] lowerValue, <span class="code-object">boolean</span> lowerInclusive[], <span class="code-object">Long</span>[] upperValue, <span class="code-object">boolean</span> upperInclusive[]);
</pre> 
                   </div>
                  </div> 
                  <p>we have:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java"><span class="code-keyword">public</span> <span class="code-keyword">static</span> Query newRangeQuery(<span class="code-object">String</span> field, <span class="code-object">long</span> lowerValue, <span class="code-object">long</span> upperValue);
<span class="code-keyword">public</span> <span class="code-keyword">static</span> Query newRangeQuery(<span class="code-object">String</span> field, <span class="code-object">long</span>[] lowerValue, <span class="code-object">long</span>[] upperValue);
</pre> 
                   </div>
                  </div> 
                  <p>PointRangeQuery is horribly complex due to these nulls and boolean arrays, and need not be. Now it only works "inclusive" and is much simpler.</p> 
                  <p>XYZPoint.newSetQuery throws IOException, just creating the query. This is very confusing and unnecessary (no i/o happens).</p> 
                  <p>LatLonPoint's bounding box query is not inclusive like the other geo. And the test does not fail!</p> 
                  <p>I also found a few missing checks here and there while cleaning up.</p> 
                 </div> 
                </div>
