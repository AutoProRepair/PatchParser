                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Cf the discussion on <a href="https://issues.apache.org/jira/browse/DERBY-3823" title="NullPointerException in stress.multi test" class="issue-link" data-issue-key="DERBY-3823"><del>DERBY-3823</del></a>.</p> 
                  <p>The enclosed repro program shows what happens. When I run it with<br> client/server and embedded respectively we see these two differing<br> results:</p> 
                  <p>client/server:</p> 
                  <p>$ java -cp .:$CLASSPATH Repo<br> Done loading data<br> executing alter<br> execp.getResultDescription: select * from t1<br> 2. PS#getMetaData: char column length is 8<br> Reexecuting ps on changed table...<br> 3. RS#getMetadata: char column length is 8<br> data:1 12345678</p> 
                  <p>dag@T61pOS:~/java/sb/apps/derby3823$ !rm<br> rm -rf DERBY3823DB</p> 
                  <p>embedded:</p> 
                  <p>dag@T61pOS:~/java/sb/apps/derby3823$ java -cp .:$CLASSPATH Repro 2<br> execp.getResultDescription: insert into t1 values(?,'aaaaa')<br> execp.getResultDescription: insert into t1 values(?,'aaaaa')<br> Done loading data<br> execp.getResultDescription: select * from t1<br> execp.getResultDescription: select * from t1<br> executing alter<br> 2. PS#getMetaData: char column length is 5<br> Reexecuting ps on changed tableh...<br> 3. RS#getMetadata: char column length is 5<br> data:1 12345678</p> 
                  <p>As we can see, the metadata results are different after the ALTER<br> TABLE. The trace from EmbedPreparedData<br> ("execp.getResultDescription:") lines (see repro-patch.diff) show that<br> after ALTER, the metadata are not refreshed on the server side.</p> 
                 </div>
