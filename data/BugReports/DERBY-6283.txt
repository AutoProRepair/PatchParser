                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The fix for <a href="https://issues.apache.org/jira/browse/DERBY-5680" title="indexStat daemon processing tables over and over even when there are no changes in the tables" class="issue-link" data-issue-key="DERBY-5680"><del>DERBY-5680</del></a> currently requires hard upgrade. A soft upgrade database with an orphaned stat row, that can be caused by something like <a href="https://issues.apache.org/jira/browse/DERBY-5681" title="When a foreign key constraint on a table is dropped, the associated statistics row for the conglomerate is not removed" class="issue-link" data-issue-key="DERBY-5681"><del>DERBY-5681</del></a>, will cause DERBY to spin eating up 100% of a cpu and possibly all the disk bandwith of the disk where the database is located.</p> 
                  <p>We should implement a fix that can be applied to soft upgraded databases if at all possible.</p> 
                  <p>In soft upgraded databases we can no use the work around of dropping the statistics as it is not available<br> to soft upgraded db's with version &lt; 10.9. The only current workaround is to disable background stats<br> completely. Dropping and recreating the suspect table may also work.</p> 
                 </div>
