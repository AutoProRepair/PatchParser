                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>There are many examples of statements failing because Derby has not implicitly created the schema associated with the current user. You don't see this if the schema is the default APP schema. But if the user is anyone other than APP, then various statements can fail. Maybe we should implicitly create a schema even if the user isn't APP. Right now, you get an error like this:</p> 
                  <p>ERROR 42Y07: Schema 'ROOT' does not exist</p> 
                  <p>The following script shows an example of this problem:</p> 
                  <p>connect 'jdbc:derby:memory:db;create=true;user=esq';</p> 
                  <p>create table licreq( domain varchar( 10 ) );</p> 
                  <p>connect 'jdbc:derby:memory:db;user=root';</p> 
                  <p>– fails<br> ALTER TABLE esq.licreq ADD COLUMN u_domain GENERATED ALWAYS AS (UPPER(domain));</p> 
                  <p>connect 'jdbc:derby:memory:db;user=app';</p> 
                  <p>– succeeds<br> ALTER TABLE esq.licreq ADD COLUMN u_domain GENERATED ALWAYS AS (UPPER(domain));</p> 
                 </div> 
                </div> 
               </div>
              </div>
