               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>In <a href="https://issues.apache.org/jira/browse/LUCENE-2309" title="Fully decouple IndexWriter from analyzers" class="issue-link" data-issue-key="LUCENE-2309"><del>LUCENE-2309</del></a> it became clear that we'd benefit a lot from Analyzer having to return reusable TokenStreams. This is a big chunk of work, but its time to bite the bullet.</p> 
                  <p>I plan to attack this in the following way:</p> 
                  <ul class="alternate" type="square"> 
                   <li>Collapse the logic of ReusableAnalyzerBase into Analyzer</li> 
                   <li>Add a ReuseStrategy abstraction to Analyzer which controls whether the TokenStreamComponents are reused globally (as they are today) or per-field.</li> 
                   <li>Convert all Analyzers over to using TokenStreamComponents. I've already seen that some of the TokenStreams created in tests need some work to be reusable (even if they aren't reused).</li> 
                   <li>Remove Analyzer.reusableTokenStream and convert everything over to using .tokenStream (which will now be returning reusable TokenStreams).</li> 
                  </ul> 
                 </div>
