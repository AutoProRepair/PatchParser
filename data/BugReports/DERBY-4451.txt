                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I see this error when I specify the columns in a different order than in the table definition. It only fails if a multi-row table constructor is used.</p> 
                  <p>ij&gt; create table t(a int, b generated always as (-a));<br> 0 rows inserted/updated/deleted<br> ij&gt; insert into t(b,a) values (default,1);<br> 1 row inserted/updated/deleted<br> ij&gt; insert into t(b,a) values (default,1), (default, 2);<br> ERROR XJ001: Java exception: '1 &gt;= 1: java.lang.ArrayIndexOutOfBoundsException'.</p> 
                  <p>And in a sane build:</p> 
                  <p>ij&gt; insert into t(b,a) values (default,1),(default,2);<br> ERROR XJ001: Java exception: 'ASSERT FAILED More columns in result column list than in base table: org.apache.derby.shared.common.sanity.AssertFailure'.</p> 
                  <p>This bug may be similar to <a href="https://issues.apache.org/jira/browse/DERBY-4448" title="ArrayIndexOutOfBoundsException when trying to override generated column" class="issue-link" data-issue-key="DERBY-4448"><del>DERBY-4448</del></a>, but the stack trace is different, and <a href="https://issues.apache.org/jira/browse/DERBY-4448" title="ArrayIndexOutOfBoundsException when trying to override generated column" class="issue-link" data-issue-key="DERBY-4448"><del>DERBY-4448</del></a> does not raise an ASSERT FAILED in sane builds.</p> 
                 </div> 
                </div> 
               </div>
