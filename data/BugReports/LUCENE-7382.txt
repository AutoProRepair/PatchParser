                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Originally reported to the mailing list: <a href="http://mail-archives.apache.org/mod_mbox/lucene-java-user/201607.mbox/%3cCAJ0VynnMAH7N7byPevTV9Htxo-Nk-B7mwUwRgP4X8gN=V4pYBg@mail.gmail.com%3e" class="external-link" rel="nofollow">http://mail-archives.apache.org/mod_mbox/lucene-java-user/201607.mbox/%3cCAJ0VynnMAH7N7byPevTV9Htxo-Nk-B7mwUwRgP4X8gN=V4pYBg@mail.gmail.com%3e</a></p> 
                  <p><a href="https://issues.apache.org/jira/browse/LUCENE-7355" title="Leverage MultiTermAwareComponent in query parsers" class="issue-link" data-issue-key="LUCENE-7355"><del>LUCENE-7355</del></a> made a change to CustomAnalyzer.createComponents() such that it uses a different AttributeFactory. <a href="https://github.com/apache/lucene-solr/commit/e92a38af90d12e51390b4307ccbe0c24ac7b6b4e#diff-b39a076156e10aa7a4ba86af0357a0feL122" class="external-link" rel="nofollow">https://github.com/apache/lucene-solr/commit/e92a38af90d12e51390b4307ccbe0c24ac7b6b4e#diff-b39a076156e10aa7a4ba86af0357a0feL122</a></p> 
                  <p>The previous default was TokenStream.DEFAULT_TOKEN_ATTRIBUTE_FACTORY which uses PackedTokenAttributeImpl while the new default is now AttributeFactory.DEFAULT_ATTRIBUTE_FACTORY which does not use PackedTokenAttributeImpl.</p> 
                  <p><a href="https://issues.apache.org/jira/secure/ViewProfile.jspa?name=thetaphi" class="user-hover" rel="thetaphi">Uwe Schindler</a> Asked me to open an issue for this.</p> 
                 </div>
