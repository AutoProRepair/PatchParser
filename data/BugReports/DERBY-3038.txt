                  <p>Since 10.3.1.4, the import procedure fails.</p>

                  <p>I'm sure there is a connection, the same which created the statement : <br> ***************************************************************************************************<br> protected boolean importTableDataCOUNTRIES() {<br> try </p> { java.sql.Statement st = conn.createStatement(java.sql.ResultSet.TYPE_SCROLL_INSENSITIVE, java.sql.ResultSet.CONCUR_UPDATABLE); String sql = ""; sql = "CALL SYSCS_UTIL.SYSCS_IMPORT_TABLE('NMSET', 'PAYS', '/tmp/lst_pays.csv', null, null, null, 1)"; st.executeUpdate(sql); return true; } 
                  <p> catch (SQLException ex) </p> { debug(ex.getMessage()); } 
                  <p> return false;<br> }<br> ***************************************************************************************************<br> The database is not being accessed by another application. The table does <br> exist. The CSV file is comma separated and strings are enclosed by ".</p> 
                  <p>Thank you for considering this issue.</p>
