                  <p>After <a href="https://issues.apache.org/jira/browse/SOLR-1889" title="Change default value of 'mm' param to depend on explicit/implicit value of q.op" class="issue-link" data-issue-key="SOLR-1889"><del>SOLR-1889</del></a> was committed, the default for DisMax "mm" parameter changes depending on q.op. Since defaultOperator=OR in example schema.xml, and no "mm" parameter is specified in the "/browse" request handler, DisMax will fallback to mm=0%. To be consistent with 3.x behavior, we should add mm=100% for "/browse" config.</p>
