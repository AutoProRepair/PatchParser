                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I can't find any way to close the IndexSearcher (and IndexReader) that<br> is being used by SpellChecker internally.</p> 
                  <p>I've worked around this issue by keeping a single SpellChecker open<br> for each index, but I'd really like to be able to close it and<br> reopen it on demand without leaking file descriptors.</p> 
                  <p>Could we add a close() method to SpellChecker that will close the<br> IndexSearcher and null the reference to it? And perhaps add some code<br> that reopens the searcher if the reference to it is null? Or would<br> that break thread safety of SpellChecker?</p> 
                  <p>The attached patch adds a close method but leaves it to the user to<br> call setSpellIndex to reopen the searcher if desired.</p> 
                 </div> 
                </div>
