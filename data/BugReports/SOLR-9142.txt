                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I indexed a dataset of 2M docs</p> 
                  <p><tt>top_facet_s</tt> has a cardinality of 1000 which is the top level facet.<br> For nested facets it has two fields <tt>sub_facet_unique_s</tt> and <tt>sub_facet_unique_td</tt> which are string and double and have cardinality 2M</p> 
                  <p>The nested query for the double field returns in the 1s mark always. The nested query for the string field takes roughly 10s to execute.</p> 
                  <div class="code panel" style="border-style: solid;border-width: 1px;">
                   <div class="codeHeader panelHeader" style="border-bottom-width: 1px;border-bottom-style: solid;">
                    <b>nested string facet</b>
                   </div>
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">q=*:*&amp;rows=0&amp;json.facet=
	{
		<span class="code-quote">"top_facet_s"</span>: {
			<span class="code-quote">"type"</span>: <span class="code-quote">"terms"</span>,
			<span class="code-quote">"limit"</span>: -1,
			<span class="code-quote">"field"</span>: <span class="code-quote">"top_facet_s"</span>,
			<span class="code-quote">"mincount"</span>: 1,
			<span class="code-quote">"excludeTags"</span>: <span class="code-quote">"ANY"</span>,
			<span class="code-quote">"facet"</span>: {
				<span class="code-quote">"sub_facet_unique_s"</span>: {
					<span class="code-quote">"type"</span>: <span class="code-quote">"terms"</span>,
					<span class="code-quote">"limit"</span>: 1,
					<span class="code-quote">"field"</span>: <span class="code-quote">"sub_facet_unique_s"</span>,
					<span class="code-quote">"mincount"</span>: 1
				}
			}
		}
	}
</pre> 
                   </div>
                  </div> 
                  <div class="code panel" style="border-style: solid;border-width: 1px;">
                   <div class="codeHeader panelHeader" style="border-bottom-width: 1px;border-bottom-style: solid;">
                    <b>nested double facet</b>
                   </div>
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">q=*:*&amp;rows=0&amp;json.facet=
	{
		<span class="code-quote">"top_facet_s"</span>: {
			<span class="code-quote">"type"</span>: <span class="code-quote">"terms"</span>,
			<span class="code-quote">"limit"</span>: -1,
			<span class="code-quote">"field"</span>: <span class="code-quote">"top_facet_s"</span>,
			<span class="code-quote">"mincount"</span>: 1,
			<span class="code-quote">"excludeTags"</span>: <span class="code-quote">"ANY"</span>,
			<span class="code-quote">"facet"</span>: {
				<span class="code-quote">"sub_facet_unique_s"</span>: {
					<span class="code-quote">"type"</span>: <span class="code-quote">"terms"</span>,
					<span class="code-quote">"limit"</span>: 1,
					<span class="code-quote">"field"</span>: <span class="code-quote">"sub_facet_unique_td"</span>,
					<span class="code-quote">"mincount"</span>: 1
				}
			}
		}
	}
</pre> 
                   </div>
                  </div> 
                  <p>I tried to dig deeper to understand why are string nested faceting that slow compared to numeric field</p> 
                  <p>Since the top facet has a cardinality of 1000 we have to calculate sub facets on each of them. Now the key difference was in the implementation of the two .</p> 
                  <p>For the string field, In <tt>FacetField#getFieldCacheCounts</tt> we call <tt>createCollectAcc</tt> with nDocs=0 and numSlots=2M . This then initializes an array of 2M. So we create a 2M array 1000 times for this one query which from what I understand makes this query slow.</p> 
                  <p>For numeric fields <tt>FacetFieldProcessorNumeric#calcFacets</tt> uses a CountSlotAcc which doesn't assign a huge array. In this query it calls <tt>createCollectAcc</tt> with numDocs=2k and numSlots=1024 .</p> 
                  <p>In string faceting, we create the 2M array because the cardinality is 2M and we use the array position as the ordinal and value as the count. If we could improve on this it would speed things up significantly? For sub-facets we know the maximum cardinality can be at max the top level bucket count.</p> 
                 </div> 
                </div> 
               </div>
