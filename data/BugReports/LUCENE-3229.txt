                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>While using Span queries I think I've found a little bug.</p> 
                  <p>With a document like this (from the TestNearSpansOrdered unit test) :</p> 
                  <p>"w1 w2 w3 w4 w5"</p> 
                  <p>If I try to search for this span query :</p> 
                  <p>spanNear([spanNear(<span class="error">[field:w3, field:w5]</span>, 1, true), field:w4], 0, true)</p> 
                  <p>the above document is returned and I think it should not because 'w4' is not after 'w5'.<br> The 2 spans are not ordered, because there is an overlap.</p> 
                  <p>I will add a test patch in the TestNearSpansOrdered unit test.<br> I will add a patch to solve this issue too.<br> Basicaly it modifies the two docSpansOrdered functions to make sure that the spans does not overlap.</p> 
                 </div>
