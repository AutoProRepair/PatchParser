               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Hello,</p> 
                  <p>The NumberUtils.createXXX methods all have the following pattern:</p> 
                  <p>public static XXX createXXX(String str) {<br> if (str == null) </p> { return null; }
                  <br> 
                  <br> In the case of BigDecimal, passing in a "" to new BigDecimal(String) in Sun
                  <br> 1.3.1_08 blows up like this:
                  <br> 
                  <br> java.lang.StringIndexOutOfBoundsException: String index out of range: 0
                  <br> at java.lang.String.charAt(String.java:582)
                  <br> at java.math.BigDecimal.&lt;init&gt;(BigDecimal.java:124)
                  <br> at org.apache.commons.lang.math.NumberUtils.createBigDecimal(NumberUtils.java:4
                  <br> 78)
                  <br> at org.apache.commons.lang.math.NumberUtilsTest.testCreateBigDecimal(NumberUtil
                  <br> sTest.java:209)
                  <br> 
                  <br> Under Sun 1.4.2, you get a NumberFormatException if the length of the string is
                  <br> 0 (no trim()). The unit tests expect a NumberFormatException when you pass in "".
                  <br> 
                  <br> So... to make this all nice on 1.3, should all of the guard clauses become:
                  <br> 
                  <br> (1)
                  <br> 
                  <br> if (StringUtil.isEmpty(str)) { return null; } 
                  <p>(2)</p> 
                  <p> if (StringUtil.isBlank(str)) </p> { return null; }
                  <br> 
                  <br> (3)
                  <br> 
                  <br> if (str == null) { return null; } 
                  <p> if (StringUtil.isEmpty(str)) </p> { return str; } 
                  <p>?</p> 
                  <p>I think (2) would be good since it would not blow up on "" AND " " but I am not<br> familiar with the various invocation contexts, so, please opine.</p> 
                  <p>Thanks,<br> Gary</p> 
                 </div> 
                </div> 
               </div>
