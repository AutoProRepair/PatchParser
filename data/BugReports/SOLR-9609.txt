                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>In order to configure our dataSource without requiring a plaintext password in the configuration file, we extended JdbcDataSource to create our own custom implementation. Our dataSource config now looks something like this:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-xml"><span class="code-tag">&lt;dataSource type=<span class="code-quote">"com.foo.FooDataSource"</span> driver=<span class="code-quote">"oracle.jdbc.OracleDriver"</span> url=<span class="code-quote">"jdbc:oracle:thin:@db-host-machine:1521:tst1"</span> user=<span class="code-quote">"testuser"</span> password=<span class="code-quote">"{ENC}{1.1}1ePOfWcbOIU056gKiLTrLw=="</span>/&gt;</span>
</pre> 
                   </div>
                  </div> 
                  <p>We are using the RSA JSAFE Crypto-J libraries for encrypting/decrypting the password. However, this seems to cause an issue when we try use Solr in a Cloud Configuration (using Zookeeper). The error is "Strong key gen and multiprime gen require at least 1024-bit keysize." Full log attached.</p> 
                  <p>This seems to be due to the hard-coded value of 512 in the org.apache.solr.util.CryptoKeys$RSAKeyPair class:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java"><span class="code-keyword">public</span> RSAKeyPair() {
  KeyPairGenerator keyGen = <span class="code-keyword">null</span>;
  <span class="code-keyword">try</span> {
    keyGen = KeyPairGenerator.getInstance(<span class="code-quote">"RSA"</span>);
  } <span class="code-keyword">catch</span> (NoSuchAlgorithmException e) {
    <span class="code-keyword">throw</span> <span class="code-keyword">new</span> SolrException(SolrException.ErrorCode.SERVER_ERROR, e);
  }
  keyGen.initialize(512);
</pre> 
                   </div>
                  </div> 
                  <p>I pulled down the Solr code, changed the hard-coded value to 1024, rebuilt it, and now everything seems to work great.</p> 
                 </div> 
                </div>
