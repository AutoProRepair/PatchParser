                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I'll link the originating Solr JIRA in a minute (many thanks Nikolay). </p> 
                  <p>right at the top of this method we have this:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">
<span class="code-object">String</span> result = dataDir + <span class="code-quote">"index/"</span>;
</pre> 
                   </div>
                  </div> 
                  <p>If, for any reason, the method doesn't complete properly, the "result" is still returned. Now for instance, down in SolrCore.cleanupOldIndexDirectories the "old" directory is dataDir/index which may point to the current index.</p> 
                  <p>This seems particularly dangerous:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">
       <span class="code-keyword">try</span> {
          p.load(<span class="code-keyword">new</span> InputStreamReader(is, StandardCharsets.UTF_8));

          <span class="code-object">String</span> s = p.getProperty(<span class="code-quote">"index"</span>);
          <span class="code-keyword">if</span> (s != <span class="code-keyword">null</span> &amp;&amp; s.trim().length() &gt; 0) {
              result = dataDir + s;
          }

        } <span class="code-keyword">catch</span> (Exception e) {
          log.error(<span class="code-quote">"Unable to load "</span> + IndexFetcher.INDEX_PROPERTIES, e);
        } <span class="code-keyword">finally</span> {
          IOUtils.closeQuietly(is);
        }

</pre> 
                   </div>
                  </div> 
                  <p>Should "p.load" fail for any reason whatsoever, we'll still return dataDir/index.</p> 
                  <p>Anyone want to chime on on what the expectations are here before I dive in?</p> 
                 </div> 
                </div> 
               </div>
