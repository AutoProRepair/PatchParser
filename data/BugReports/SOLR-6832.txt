                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Currently, I see that code flow for a query in SolrCloud is as follows:</p> 
                  <p>For distributed query:<br> SolrCore -&gt; SearchHandler.handleRequestBody() -&gt; HttpShardHandler.submit()</p> 
                  <p>For non-distributed query:<br> SolrCore -&gt; SearchHandler.handleRequestBody() -&gt; QueryComponent.process()</p> 
                  <p><br class="atl-forced-newline"> <br class="atl-forced-newline"> <br class="atl-forced-newline"></p> 
                  <p>For a distributed query, the request is always sent to all the shards even if the originating SolrCore (handling the original distributed query) is a replica of one of the shards.<br> If the original Solr-Core can check itself before sending http requests for any shard, we can probably save some network hopping and gain some performance.</p> 
                  <p><br class="atl-forced-newline"> <br class="atl-forced-newline"></p> 
                  <p>We can change SearchHandler.handleRequestBody() or HttpShardHandler.submit() to fix this behavior (most likely the former and not the latter).</p> 
                 </div> 
                </div> 
               </div>
