                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The following script and output shows the problem:</p> 
                  <p>&gt; create table r(c1 char(1));<br> &gt; create table s(c1 char(1), c2 char(1));<br> &gt; create table t(c1 char(1));</p> 
                  <p>&gt; insert into r values 'a';<br> &gt; insert into s values ('b', default);<br> &gt; insert into t values ('c');</p> 
                  <p>&gt; select * from s left outer join t on s.c2=t.c1 or s.c2 is null;</p> 
                  <p>C1 |C2 |C1 <br> --------------<br> b |NULL|c </p> 
                  <p>&gt; select * from r left outer join s on r.c1=s.c1;<br> C1 |C1 |C2 <br> --------------<br> a |NULL|NULL</p> 
                  <p>&gt; select * from (r left outer join s on r.c1=s.c1) left outer join t on s.c2=t.c1 or s.c2 is null;</p> 
                  <p>C1 |C1 |C2 |C1 <br> -------------------<br> a |NULL|NULL|c </p> 
                  <p>&gt; select * from r left outer join (s left outer join t on s.c2=t.c1 or s.c2 is null) on r.c1=s.c1;</p> 
                  <p>C1 |C1 |C2 |C1 <br> -------------------<br> a |NULL|NULL|c </p> 
                  <p>The last result is wrong. The correct answer should be:</p> 
                  <p>C1 |C1 |C2 |C1 <br> -------------------<br> a |NULL|NULL|NULL </p> 
                  <p>since in the last form, the left table r has the value 'a', which does<br> not match any row in result of the compound inner given the join<br> predicate ("r.c1=s.c1"), so all nulls should be appended to the 'a'<br> from the outer table r.</p> 
                  <p>This happens because internally the last form is rewritten to the<br> second but the last form (left-deep), but this rewrite is not<br> justified here unless the join predicate on s rejects null, which the<br> present one explicitly does not ("or s.c2 is null"). Cf. for example<br> <span class="error">[1]</span>, page 52, which describes this transform and its prerequisite<br> condition as indentity #7.</p> 
                  <p><span class="error">[1]</span> Galindo-Legaria, C. &amp; Rosenthal, A.: "Outerjoin simplification and<br> reordering for query optimization", ACM Transactions on Database<br> Systems, Vol 22, No 1, March 1997.</p> 
                 </div> 
                </div> 
               </div>
