                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The version is really "just" 3 (maybe 4) ints under-the-hood, but today we write it as a String which then requires spooky string tokenization/parsing when we open the index. I think it should be encoded directly as ints.</p> 
                  <p>In <a href="https://issues.apache.org/jira/browse/LUCENE-5952" title="Give Version parsing exceptions more descriptive error messages" class="issue-link" data-issue-key="LUCENE-5952"><del>LUCENE-5952</del></a> I had tried to make this change, but it was controversial, and got booted.</p> 
                  <p>Then in <a href="https://issues.apache.org/jira/browse/LUCENE-5969" title="Add Lucene50Codec" class="issue-link" data-issue-key="LUCENE-5969"><del>LUCENE-5969</del></a>, I tried again, but that issue has morphed (nicely!) into fixing all sorts of things <b>except</b> these three ints.</p> 
                  <p>Maybe 3rd time's a charm <img class="emoticon" src="/jira/images/icons/emoticons/wink.png" height="16" width="16" align="absmiddle" alt="" border="0"></p>
