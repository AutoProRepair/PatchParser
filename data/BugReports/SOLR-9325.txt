                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>(6.1 is probably also affected, but we've been blocked by <a href="https://issues.apache.org/jira/browse/SOLR-9231" title="SolrInputDocument no-args constructor removed without notice" class="issue-link" data-issue-key="SOLR-9231">SOLR-9231</a>)</p> 
                  <p>solr.log should be written to the directory specified by the SOLR_LOGS_DIR environment variable, but instead it's written to </p> {solrRoot} 
                  <p>/server/logs.</p> 
                  <p>This results in requiring that solr is installed on a writable device, which leads to two problems:<br> 1) solr installation can't live on a shared device (single copy shared by two or more VMs)<br> 2) solr installation is more difficult to lock down</p> 
                  <p>Solr should be able to run without error in this test scenario:</p> 
                  <p>burn the Solr directory tree onto a CD-ROM<br> Mount this CD as /solr<br> run Solr from there (with appropriate environment variables set, of course)</p> 
                 </div> 
                </div> 
               </div>
