                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Spinoff from here: <a href="http://lucene.472066.n3.nabble.com/TestUtil-getTempFile-may-fail-on-quot-Access-Denied-quot-td4028048.html" class="external-link" rel="nofollow">http://lucene.472066.n3.nabble.com/TestUtil-getTempFile-may-fail-on-quot-Access-Denied-quot-td4028048.html</a>.</p> 
                  <p>_TestUtil.getTempDir uses createTempFile and then deletes the file. While this usually works, if someone runs tests by multiple JVMs and does not ensure each JVM gets an isolated temp.dir to work in, that my result in two JVMs sharing the same directory.</p> 
                  <p>Also, on Windows, if you call getTempDir on an existing directory, you get an "Access is denied" exception.</p> 
                  <p>Dawid proposed a simple solution to just call mkdirs() continuously until success. I'd like to try that.</p> 
                  <p>Also, I think that genTempFile could use some house cleaning, e.g.:</p> 
                  <ul> 
                   <li>tempFileLocker can be just an Object instance? Why do we need a class?</li> 
                   <li>If we initialize counter and counterBase in a static clause, we can avoid checking if counter==0 as well as passing Random to genTempFile (that will remove any suspicion that it does anything randomly) 
                    <ul> 
                     <li>Also, instead of synchronizing on tempFileLocker, can we just use AtomicInteger for the counter?</li> 
                    </ul> </li> 
                  </ul> 
                  <p>I'll modify getTempDir first. It documents "does not create the directory", I want to make sure no test fails due that.</p> 
                 </div> 
                </div> 
               </div>
