                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>This happens from JDBC or ij. Here the output form ij&gt;</p> 
                  <p>ij version 10.0 <br> CONNECTION0* - jdbc:derby:phsDB </p> 
                  <ul> 
                   <li>= current connection<br> ij&gt; select a1.XXX_foreign, a1.native, a1.kind, a1.XXX_foreign FROM slg_name_lookup a1 ORDER BY a1.XXX_foreign;<br> ERROR 42X79: Column name 'XXX_FOREIGN' appears more than once in the result of the query expression. </li> 
                  </ul> 
                  <p>But when removing the ORDER BY and keeping the 2 same column names it works</p> 
                  <p>ij&gt; select a1.XXX_foreign, a1.native, a1.kind, a1.XXX_foreign FROM slg_name_lookup a1;<br> XXX_FOREIGN |NATIVE |KIND |XXX_FOREIGN ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- <br> 0 rows selected <br> ij&gt; </p> 
                  <p>So - it seams to be OK to specify the same column twice - as long as you do not add the ORDER BY clause. </p> 
                  <p>I woul dof course like that the system allows this - but at leats it should be consistant and either allow both or none of the two queries above.</p> 
                 </div> 
                </div>
