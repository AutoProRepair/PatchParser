                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>If using the Derby Client to connect to the Derby Network Server, there are some cases where an invalid URL will cause a protocol exception instead of throwing a proper error. The two cases I've found are as follow:</p> 
                  <p>1) Use of a colon to separate attributes:</p> 
                  <p>When connecting to the server with JCC, a colon is required to demarcate the Derby attributes from the JCC attributes. For the Derby Client, however, the colon is not a valid part of the URL, so use of it should return an error. However, the result is a protocol exception:</p> 
                  <p>ij&gt; connect 'jdbc:derby://localhost:1528/sample:user=someUsr;password=somePwd';<br> agentThread<span class="error">[DRDAConnThread_9,5,main]</span><br> ERROR (no SQLState): A communication error has been detected. Communication protocol being used: Reply.fill(). Communication API being used: InputStream.read(). Location where the error was detected:<br> insufficient data. Communication function detecting the error: *. Protocol specific error codes(s) TCP/IP SOCKETS</p> 
                  <p>2) Use of double-quotes to specify a full path to a database.</p> 
                  <p>The use of double-quotes around a database path is required when connecting to the server with JCC, but should be optional when using the Derby Client. However, attempts to use the double-quotes doesn't work--on the contrary, it throws a protocol exception:</p> 
                  <p>ij&gt; connect 'jdbc:derby://localhost:1528/"C:/myDBs/sample"';<br> agentThread<span class="error">[DRDAConnThread_11,5,main]</span><br> ERROR (no SQLState): A communication error has been detected. Communication protocol being used: Reply.fill(). Communication API being used: InputStream.read(). Location where the error was detected:<br> insufficient data. Communication function detecting the error: *. Protocol specific error codes(s) TCP/IP SOCKETS</p> 
                  <p>Without quotes, the above example will work as expected.</p> 
                  <p>I'm not sure if the problem is with the Derby client or the Network Server (more investigation required).</p> 
                 </div>
