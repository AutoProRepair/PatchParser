               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The sequence below shows that a query returns 1 row when there is no index on the table, and it returns 0 rows when an index is created. It should return 1 row regardless of the index's presence.</p> 
                  <p>ij version 10.9<br> ij&gt; connect 'jdbc:derby:memory:colldb;create=true;territory=no;collation=TERRITORY_BASED';<br> ij&gt; create table t(x varchar(40));<br> 0 rows inserted/updated/deleted<br> ij&gt; insert into t values 'Stranda Idrottslag', 'Aalesunds Fotballklubb';<br> 2 rows inserted/updated/deleted<br> ij&gt; select * from t where x in ('Aalesunds Fotballklubb', cast('xyz' as char(3)));<br> X <br> ----------------------------------------<br> Aalesunds Fotballklubb </p> 
                  <p>1 row selected<br> ij&gt; create index i on t<img class="emoticon" src="/jira/images/icons/emoticons/error.png" height="16" width="16" align="absmiddle" alt="" border="0">;<br> 0 rows inserted/updated/deleted<br> ij&gt; select * from t where x in ('Aalesunds Fotballklubb', cast('xyz' as char(3)));<br> X <br> ----------------------------------------</p> 
                  <p>0 rows selected</p> 
                 </div> 
                </div>
