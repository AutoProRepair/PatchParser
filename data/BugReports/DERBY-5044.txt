                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>A trigger in it's trigger action.can use columns from a table other than the trigger table. When such a column is dropped, the trigger dependency does not get detected.</p> 
                  <p>A test case for this can be found in AlterTableTest.java<br> //Following test case involves two tables. The trigger is defined <br> //on table 1 and it uses the column from table 2 in it's trigger <br> //action. This dependency of the trigger on a column from another <br> //table is not detected by Derby.<br> st.executeUpdate("create table atdc_14_tab1 (a1 integer, b1 integer)");<br> st.executeUpdate("create table atdc_14_tab2 (a2 integer, b2 integer)"); <br> st.executeUpdate("insert into atdc_14_tab1 values(1,11)");<br> st.executeUpdate("insert into atdc_14_tab2 values(1,11)");<br> st.executeUpdate(<br> " create trigger atdc_14_trigger_1 after update " +<br> "on atdc_14_tab1 REFERENCING NEW AS newt " +<br> "for each row " +<br> "update atdc_14_tab2 set a2 = newt.a1");</p> 
                  <p> // following is not the right behavior. we should have gotten an error<br> // because column being dropped is getting used in a trigger action <br> st.executeUpdate("alter table atdc_14_tab2 drop column a2 restrict");<br> rs =<br> st.executeQuery(<br> " select triggername from sys.systriggers where " +<br> "triggername = 'ATDC_14_TRIGGER_1' ");<br> JDBC.assertFullResultSet(rs, new String[][]<tt>"ATDC_14_TRIGGER_1"</tt>);</p> 
                 </div> 
                </div> 
               </div>
