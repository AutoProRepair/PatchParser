                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Possibly related to #4332?</p> 
                  <p>We have encountered an error when using Prepared Statements and CLOBs. I have read:</p> 
                  <p><a href="http://db.apache.org/derby/papers/JDBCImplementation.html#setAsciiStream%2CsetBinaryStream%2CsetCharacterStream" class="external-link" rel="nofollow">http://db.apache.org/derby/papers/JDBCImplementation.html#setAsciiStream%2CsetBinaryStream%2CsetCharacterStream</a></p> 
                  <p>But it does not seem applicable, as we are not re-using a stream.</p> 
                  <p>The environment is this:</p> 
                  <p>1. Java 6<br> 2. Derby 10.5.3.0<br> 3. Bitronix JTA 1.3.3</p> 
                  <p>We're actually using Hibernate, but I eliminated it from the equation (and the problem persists).</p> 
                  <p>A summary of the failure flow is this:</p> 
                  <p>1. Start a transaction<br> 2. Obtain a connection from a pool of connections (for this test, the pool size is pinned at 1)<br> 3. Prepare a statement that inserts a CLOB.<br> 4. Set the parameters<br> 5. Add the prepared statement to a batch (but we only batch 1 – this is to emulate what hibernate is doing as closely as possible).<br> 6. Execute the batch.</p> 
                  <p>Everything up to this point works.</p> 
                  <p>7. Repeat steps 1-6. But this time, the connection will be reused from the pool, and the statement will be gotten from a prepared statement cache (maintained by bitronix). I.e. the prepared statement is re-used.<br> 8. Observe the following failure:</p>
