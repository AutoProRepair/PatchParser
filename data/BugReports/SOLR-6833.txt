                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>i think it's weird right now that running bin/solr with the "-e" (example) option causes it to create example solr instances inside the server directory.</p> 
                  <p>i think that's fine for running solr "normally" (ie: "start") but if you use "-e" that seems like the solr.solr.home for those example should instead be created under $SOLR_TIP/example.</p> 
                  <p>I would even go so far as to suggest that the <b>log</b> files created should live in that directory as well.</p> 
                 </div> 
                </div>
