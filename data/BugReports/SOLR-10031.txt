                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Fra: Mark Thomas &lt;markt@apache.org&gt;<br> Emne: Fwd: Apache Solr - security vulnerability (path traversal attack)<br> Dato: 24. januar 2017 kl. 13.14.36 CET<br> Til: private@lucene.apache.org<br> Kopi: "security@apache.org" &lt;security@apache.org&gt;<br> Svar til: private@lucene.apache.org</p> 
                  <p>Dear Apache Lucene PMC,</p> 
                  <p>The security vulnerability report has been received by the Apache<br> Security Team and is being passed to you for action.</p> 
                  <p>Please take careful note of the following:</p> 
                  <ul class="alternate" type="square"> 
                   <li>This information is private and should be treated accordingly. The<br> issue must not be discussed on a public mailing list, it must not be<br> added to a public bug tracker, etc.</li> 
                   <li>The Lucene PMC is responsible for resolving this issue. The security<br> team is here to provide help and advice but the responsibility to do the<br> work lies with the Lucene PMC.</li> 
                  </ul> 
                  <p>You may find the "ASF Project Security for Committers" <span class="error">[1]</span> a useful<br> reference. This e-mail represents step three of that process. Step 4<br> should be completed asap.</p> 
                  <p>Kind regards,</p> 
                  <p>Mark</p> 
                  <p><span class="error">[1]</span> <a href="http://www.apache.org/security/committers.html" class="external-link" rel="nofollow">http://www.apache.org/security/committers.html</a></p> 
                  <p>-------- Forwarded Message --------<br> Subject: Apache Solr - security vulnerability (path traversal attack)<br> Date: Mon, 23 Jan 2017 11:27:19 -0800<br> From: Hrishikesh Gadre &lt;gadre.solr@gmail.com&gt;<br> To: security@apache.org<br> CC: Hrishikesh Gadre &lt;gadre.solr@gmail.com&gt;</p> 
                  <p>Hi,</p> 
                  <p>We found a path manipulation security vulnerability in Apache Solr after<br> running HPE Fortify static code analyzer on the Solr codebase.</p> 
                  <p>Here is a brief description of this issue,</p> 
                  <ul class="alternate" type="square"> 
                   <li>Apache Solr provides a "replication" handler which supports operations<br> related to querying the state of an index as well as copying files<br> associated with the index.</li> 
                  </ul> 
                  <p><a href="https://cwiki.apache.org/confluence/display/solr/Index+Replication" class="external-link" rel="nofollow">https://cwiki.apache.org/confluence/display/solr/Index+Replication</a><br> &lt;<a href="https://cwiki.apache.org/confluence/display/solr/Index+Replication" class="external-link" rel="nofollow">https://cwiki.apache.org/confluence/display/solr/Index+Replication</a>&gt;</p> 
                  <p>This handler supports an HTTP API<br> (/replication?command=filecontent&amp;file=&lt;file_name&gt;) which is vulnerable<br> to path traversal attack. Specifically, this API does not perform any<br> validation of the user specified file_name parameter. This can allow an<br> attacker to download <b>any</b> file readable to Solr server process even if<br> it is not related to the actual Solr index state.<br> <a href="https://www.owasp.org/index.php/Path_Traversal" class="external-link" rel="nofollow">https://www.owasp.org/index.php/Path_Traversal</a></p> 
                  <p>I have verified this with the Solr version 6.3. But I believe this<br> vulnerability to be present for much longer (going back to v 4.10.x) . I<br> am currently working on the fix. Please let me know the process to<br> submit a patch for this.</p> 
                  <p>Thanks<br> Hrishikesh</p>
