                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>When trying to insert a row into the table </p> 
                  <p>SHORT_TEST( ID int, SHORT_VAL smallint)</p> 
                  <p>an exception is thrown, if the object value given to PreparedStatement#setObject(int parameterIndex, Object x) is of type Short. The exception thrown is:</p> 
                  <p>— SQLException —<br> SQLState: 22005<br> Message: An attempt was made to get a data value of type 'SMALLINT' from a data value of type 'java.lang.Short'.<br> ErrorCode: 20000<br> SQL Exception: An attempt was made to get a data value of type 'SMALLINT' from a data value of type 'java.lang.Short'.<br> at org.apache.derby.impl.jdbc.Util.newEmbedSQLException(Unknown Source)<br> at org.apache.derby.impl.jdbc.Util.newEmbedSQLException(Unknown Source)<br> at org.apache.derby.impl.jdbc.Util.generateCsSQLException(Unknown Source)<br> at org.apache.derby.impl.jdbc.EmbedConnection.newSQLException(Unknown Source)<br> at org.apache.derby.impl.jdbc.ConnectionChild.newSQLException(Unknown Source)<br> at org.apache.derby.impl.jdbc.EmbedPreparedStatement.dataTypeConversion(Unknown Source)<br> at org.apache.derby.impl.jdbc.EmbedPreparedStatement.setObject(Unknown Source)</p> 
                  <p>Tested on Derby 10.1.1.0 and 10.1.3.1. The same test runs fine in network mode.</p> 
                 </div> 
                </div> 
               </div>
