                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The current tree data structure in GeoComplexPolygon has a lot of weaknesses. A better algorithm maybe can be taken from Polygon2D, which basically does the following:</p> 
                  <p>Each node has:</p> 
                  <ul class="alternate" type="square"> 
                   <li>low value (which is for that edge alone)</li> 
                   <li>max value (which is for that edge and all children)</li> 
                  </ul> 
                  <p>Balanced tree building:</p> 
                  <ul class="alternate" type="square"> 
                   <li>sort by low value (which is for the individual edge), and use max value as tie breaker (which is max for edge and all children)</li> 
                   <li>build tree after sorting, picking midpoint and recursively building lesser and greater children that way</li> 
                  </ul> 
                  <p>Balanced tree traversal (looking for range minValue -&gt; maxValue):</p> 
                  <ul class="alternate" type="square"> 
                   <li>Descend the entire tree until the node fails this test:<br> if (minValue &lt;= max) { ... } <p> So if the minimum value being sought is greater than the max for this edge and all children, we stop and don't look at children.<br> (Q: does this represent a good split and a targeted range? Maybe... We can certainly try it.)</p></li> 
                  </ul> 
                 </div> 
                </div>
