                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The test run for 10.5.2.0 hung in suites.All. The console output (the run was with -Dderby.tests.trace=true) showed ttestDefaultProperties had successfully completed but the run was halted.<br> ps -eaf | grep java showed the process that kicked off suites.All, and a networkserver process with the following flags:</p> 
                  <ul class="alternate" type="square"> 
                   <li>classpath &lt;classpath including derby.jar, derbytools.jar, derbyclient.jar, derbynet.jar, derbyTesting.jar, derbyrun.jar, derbyTesting.jar and junit.jar&gt; -Dderby.drda.logConnections= -Dderby.drda.traceAll= -Dderby.drda.traceDirectory= -Dderby.drda.keepAlive= -Dderby.drda.timeSlice= -Dderby.drda.host= -Dderby.drda.portNumber= -derby.drda.minThreads= -Dderby.drda.maxThreads= -Dderby.drda.startNetworkServer= -Dderby.drda.debug= org.apache.derby.drda.NetworkServerControl start -h localhost -p 1527<br> This process had been sitting for 2 days.<br> After killing the NetworkServerControl process, the test continued successfully (except for <a href="https://issues.apache.org/jira/browse/DERBY-4186" title="After master stop, test fails when it succeeds in connecting (rebooting) shut-down ex-slave" class="issue-link" data-issue-key="DERBY-4186"><del>DERBY-4186</del></a>, fixed in trunk), but the following was put out to the console:<br> START-SPAWNED:SpawnedNetworkServer STANDARD OUTPUT: exit code=137<br> 2009-07-18 03:16:07.157 GMT : Security manager installed using the Basic server<br> security policy.<br> 2009-07-18 03:16:09.169 GMT : Apache Derby Network Server - 10.5.2.0 - (794445)<br> started and ready to accept connections on port 1527<br> END-SPAWNED :SpawnedNetworkServer STANDARD OUTPUT:</li> 
                  </ul> 
                 </div>
