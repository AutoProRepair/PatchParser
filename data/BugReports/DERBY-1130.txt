                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Per this thread, setConnectionAttributes should not set databaseName. </p> 
                  <p><a href="http://www.nabble.com/double-check-on-checkDataSource-t1187602.html#a3128621" class="external-link" rel="nofollow">http://www.nabble.com/double-check-on-checkDataSource-t1187602.html#a3128621</a></p> 
                  <p>Currently this is allowed for client but should be disabled. I think it is OK to change because we have documented that client will be changed to match embedded for implementation defined behaviour. Hopefully its use is rare as most folks would use the standard setDatabaseName. Still there should be a release not when the change is made and it would be better to change it sooner than later:</p> 
                  <p>Below is the repro. </p> 
                  <p>Here is the output with Client<br> D&gt;java DatabaseNameWithSetConnAttr<br> ds.setConnectionAttributes(databaseName=wombat;create=true)<br> ds.getDatabaseName() = null (should be null)<br> FAIL: Should not have been able to set databaseName with connection attributes</p> 
                  <p>Also look for tests disabled with this bug number in the test checkDataSource30.java</p> 
                  <p>import java.sql.*;<br> import java.lang.reflect.Method;</p> 
                  <p>public class DatabaseNameWithSetConnAttr{</p> 
                  <p> public static void main(String[] args) {<br> try </p> { String attributes = "databaseName=wombat;create=true"; org.apache.derby.jdbc.ClientDataSource ds = new org.apache.derby.jdbc.ClientDataSource(); //org.apache.derby.jdbc.EmbeddedDataSource ds = new //org.apache.derby.jdbc.EmbeddedDataSource(); System.out.println("ds.setConnectionAttributes(" + attributes + ")"); ds.setConnectionAttributes(attributes); System.out.println("ds.getDatabaseName() = " + ds.getDatabaseName() + " (should be null)" ); Connection conn = ds.getConnection(); } 
                  <p> catch (SQLException e) {<br> String sqlState = e.getSQLState();<br> if (sqlState != null &amp;&amp; sqlState.equals("XJ041"))</p> { System.out.println("PASS: An exception was thrown trying to get a connetion from a datasource after setting databaseName with setConnectionAttributes"); System.out.println("EXPECTED EXCEPTION: " + e.getSQLState() + " - " + e.getMessage()); return; } 
                  <p> while (e != null)</p> { System.out.println("FAIL - UNEXPECTED EXCEPTION: " + e.getSQLState()); e.printStackTrace(); e = e.getNextException(); } 
                  <p> return;<br> }<br> System.out.println("FAIL: Should not have been able to set databaseName with connection attributes");</p> 
                  <p> }</p> 
                  <p>}</p> 
                 </div> 
                </div>
