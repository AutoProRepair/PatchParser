                 <div class="user-content-block"> 
                  <p>The attached program ScanPosSaved.java produces the error below, fairly quickly. <br> The program has three threads, one doing inserts into a table with an identity column, one updating the row with the maximum id, one deleting the row with the maximum id.</p> 
                  <p>To reproduce, run &gt;java ScanPosSaved and &lt;ctrl&gt; &lt;c&gt; out of the program after you get the error.</p> 
                  <p>I saw this 10.5 and trunk sane builds but did not see it on 10.4. With the insane build of 10.5.1.1 (RC2) I did not see any symptoms right away, so don't know how serious an issue this is for insane builds.</p>
