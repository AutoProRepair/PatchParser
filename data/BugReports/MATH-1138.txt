                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I have encountered a use case with the BicubicSplineInterpolator where the interpolated values that are being returned seem incorrect. Furthermore, the values do not match those generated by MatLab using the interp2 'cubic' method.</p> 
                  <p>Here is a snippet of code that uses the interpolator:</p> 
                  <p> double[] xValues = new double[] </p> {36, 36.001, 36.002} 
                  <p>;<br> double[] yValues = new double[] {-108.00, -107.999, -107.998};</p> 
                  <p> double[][] fValues = new double[][] {{1915, 1906, 1931},<br> </p> {1877, 1889, 1894} 
                  <p>,<br> {1878, 1873, 1888}};</p> 
                  <p> BicubicSplineInterpolator interpolator = new BicubicSplineInterpolator();<br> BicubicSplineInterpolatingFunction interpolatorFunction = interpolator.interpolate(xValues, yValues, fValues);</p> 
                  <p> double[][] results = new double<span class="error">[9]</span><span class="error">[9]</span>;<br> double x = 36;<br> int arrayIndexX = 0, arrayIndexY = 0;</p> 
                  <p> while(x &lt;= 36.002) {<br> double y = -108;<br> arrayIndexY = 0;<br> while (y &lt;= -107.998) </p> { results[arrayIndexX][arrayIndexY] = interpolatorFunction.value(x, y); System.out.println(results[arrayIndexX][arrayIndexY]); y = y + 0.00025; arrayIndexY++; } 
                  <p> x = x + 0.00025;<br> arrayIndexX++;<br> }</p> 
                  <p>Attached is a grid showing x and y values and the corresponding interpolated value from both commons math and MatLab.</p> 
                  <p>The values produced by commons math are far off from those created by MatLab.</p>
