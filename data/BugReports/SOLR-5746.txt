                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>A comment in the ref guide got me looking at ConfigSolrXml.java and noticing that the parsing of solr.xml options here is very brittle and confusing. In particular:</p> 
                  <ul> 
                   <li>if a boolean option "foo" is expected along the lines of <tt>&lt;bool name="foo"&gt;true&lt;/bool&gt;</tt> it will silently ignore <tt>&lt;str name="foo"&gt;true&lt;/str&gt;</tt></li> 
                   <li>likewise for an int option <tt>&lt;int name="bar"&gt;32&lt;/int&gt;</tt> vs <tt>&lt;str name="bar"&gt;32&lt;/str&gt;</tt></li> 
                  </ul> 
                  <p>... this is inconsistent with the way solrconfig.xml is parsed. In solrconfig.xml, the xml nodes are parsed into a NamedList, and the above options will work in either form, but an invalid value such as <tt>&lt;bool name="foo"&gt;NOT A BOOLEAN&lt;/bool&gt;</tt> will generate an error earlier (when parsing config) then <tt>&lt;str name="foo"&gt;NOT A BOOLEAN&lt;/str&gt;</tt> (attempt to parse the string as a bool the first time the config value is needed)</p> 
                  <p>In addition, i notice this really confusing line...</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">    propMap.put(CfgProp.SOLR_SHARESCHEMA, doSub(<span class="code-quote">"solr/str[@name=<span class="code-quote">'shareSchema'</span>]"</span>));
</pre> 
                   </div>
                  </div> 
                  <p>"shareSchema" is used internally as a boolean option, but as written the parsing code will ignore it unless the user explicitly configures it as a <tt>&lt;str/&gt;</tt></p> 
                 </div> 
                </div> 
               </div>
