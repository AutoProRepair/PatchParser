                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Stanislav Bryzgalov reported that the following script raises an NPE on the last query (a left join involving an aggregate). I have verified this in my environment:</p> 
                  <p>drop table t1;<br> drop table t2;</p> 
                  <p>– create two simple tables<br> CREATE TABLE T1( D1 DATE NOT NULL PRIMARY KEY, N1 VARCHAR( 10 ) );<br> CREATE TABLE T2( D2 DATE NOT NULL PRIMARY KEY, N2 VARCHAR( 10 ) );</p> 
                  <p>– insert some data, two recs in T1 and one in T2<br> INSERT INTO T1 VALUES( DATE( '2008-10-01' ), 'something' ), ( DATE( '2008-10-02' ), 'something' );<br> INSERT INTO T2 VALUES( DATE( '2008-10-01' ), 'something' );</p> 
                  <p>– this runs fine, gives one record '2008-10-02'<br> SELECT T1.D1<br> FROM T1<br> LEFT JOIN T2<br> ON T1.D1 = T2.D2<br> WHERE T2.D2 IS NULL;</p> 
                  <p>– this runs fine too, gives one record '2008-10-02' <br> SELECT MAX( T1.D1 ) as D<br> FROM T1<br> WHERE T1.D1 NOT IN ( SELECT T2.D2 FROM T2 );</p> 
                  <p>– this one breaks!!!<br> – SQL State = XJ001 SQL Code = -1 SQL Message = DERBY SQL error: SQLCODE: -1, SQLSTATE: XJ001, SQLERRMC: java.lang.NullPointerException<br> SELECT MAX( T1.D1 ) AS D<br> FROM T1<br> LEFT JOIN T2<br> ON T1.D1 = T2.D2<br> WHERE T2.D2 IS NULL;</p> 
                 </div> 
                </div> 
               </div>
