                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>This method returns number of "incoming" bytes IW has written since it<br> was opened, excluding merging.</p> 
                  <p>It tracks flushed segments, new commits (segments_N), incoming<br> files/segments by addIndexes, newly written live docs / doc values<br> updates files.</p> 
                  <p>It's an easy statistic for IW to track and should be useful to help<br> applications more intelligently set defaults for IO throttling<br> (RateLimiter).</p> 
                  <p>For example, an application that does hardly any indexing but finally<br> triggered a large merge can afford to heavily throttle that large<br> merge so it won't interfere with ongoing searches.</p> 
                  <p>But an application that's causing IW to write new bytes at 50 MB/sec<br> must set a correspondingly higher IO throttling otherwise merges will<br> clearly fall behind.</p> 
                 </div> 
                </div>
