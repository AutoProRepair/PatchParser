                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I have an AFTER DELETE trigger on an exchange rate table which inserts data into a history table. It uses a sequence for the key in the history table.</p> 
                  <p>Simplified it looks like this:<br> CREATE TRIGGER TRG_EXCHANGE_RATES_HIST_DEL<br> AFTER DELETE ON EXCHANGE_RATES<br> REFERENCING OLD ROW AS OLD<br> FOR EACH ROW<br> INSERT INTO EXCHANGE_RATE_HISTORY (ID_HISTORY, CUR_FROM, AMNT_FROM, CUR_TO, AMNT_TO, AMNT_RATE, DAT_CREATION)<br> VALUES (( NEXT VALUE for HIST_SEQ ), OLD.CUR_FROM, OLD.AMNT_FROM, OLD.CUR_TO, OLD.AMNT_TO, OLD.AMNT_RATE, CURRENT_TIMESTAMP);</p> 
                  <p>The sequence is defined as:<br> create sequence HIST_SEQ<br> increment by 1<br> start with 10000001<br> no maxvalue<br> minvalue 10000001<br> no cycle;</p> 
                  <p>Randomly when deleting data the statement will fail with:<br> ERROR XSTA2: A transaction was already active, when attempt was made to make another transaction active.</p> 
                  <p>I will attach the full stack trace. It looks like the sequence cache needs to be cleared which causes a nested user transaction to start. But my delete statement is already running as part of a transaction (non XA - spring managed - hibernate).</p> 
                  <p>We do not have this exception when testing with derby 10.8.2.2.</p> 
                  <p>I get the same exception after building and running the latest trunk version (revision 1463340, Bundle-Version: 10.11.0000000.1463340)</p> 
                  <p>To rule out the sequence on the history table, I switched it to use a generated column (GENERATED ALWAYS AS IDENTITY (START WITH 10000001, INCREMENT BY 1)) but I still randomly get the exception.</p> 
                  <p>All tables in the application use sequences for their primary keys (through Hibernate's SequenceHiLoGenerator) so it seems to be possibly related to that.</p> 
                 </div> 
                </div> 
               </div>
