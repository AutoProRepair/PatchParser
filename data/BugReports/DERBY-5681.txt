                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>If you drop the foreign key constraint for a table, the statistics row does not get removed. This affects the indexStat daemon because it now finds these statistics row which always appear as out of date, causing an update to be scheduled.</p> 
                  <p>Here is how to get it to happen:</p> 
                  <p>set schema app;</p> 
                  <p>CREATE TABLE TEST_TAB_1<br> (<br> ID INTEGER PRIMARY KEY NOT NULL<br> );</p> 
                  <p>CREATE TABLE TEST_TAB_2<br> (<br> ID INTEGER PRIMARY KEY NOT NULL<br> );</p> 
                  <p>ALTER TABLE TEST_TAB_2<br> ADD CONSTRAINT TEST_TAB_2_FK_1<br> FOREIGN KEY (ID) REFERENCES TEST_TAB_1(ID);</p> 
                  <p>insert into app.TEST_TAB_1 values (1);<br> insert into test_tab_2 values(1);</p> 
                  <p>call syscs_util.syscs_update_statistics('APP', 'TEST_TAB_2', null);</p> 
                  <p>select<br> c.TABLEID,<br> c.CONGLOMERATENUMBER,<br> c.CONGLOMERATENAME,<br> c.ISINDEX,<br> c.ISCONSTRAINT,<br> c.CONGLOMERATEID,<br> t.TABLEID,<br> t.TABLENAME,<br> t.TABLETYPE,<br> s.STATID,<br> s.REFERENCEID,<br> s.TABLEID,<br> s.CREATIONTIMESTAMP,<br> s.TYPE,<br> s.VALID,<br> s.COLCOUNT,<br> CAST(STATISTICS AS VARCHAR(40)) as STATISTICS<br> from sys.SYSCONGLOMERATES c join sys.SYSTABLES t on c.TABLEID = t.TABLEID join sys.SYSSTATISTICS s on s.TABLEID = t.TABLEID<br> where t.TABLENAME = 'TEST_TAB_2' and c.ISINDEX = false;</p> 
                  <p>– At this point there are two statistic rows</p>

                  <p>– Now drop the constraint</p> 
                  <p>alter table TEST_TAB_2<br> drop constraint TEST_TAB_2_FK_1;</p> 
                  <p>select<br> c.TABLEID,<br> c.CONGLOMERATENUMBER,<br> c.CONGLOMERATENAME,<br> c.ISINDEX,<br> c.ISCONSTRAINT,<br> c.CONGLOMERATEID,<br> t.TABLEID,<br> t.TABLENAME,<br> t.TABLETYPE,<br> s.STATID,<br> s.REFERENCEID,<br> s.TABLEID,<br> s.CREATIONTIMESTAMP,<br> s.TYPE,<br> s.VALID,<br> s.COLCOUNT,<br> CAST(STATISTICS AS VARCHAR(40)) as STATISTICS<br> from sys.SYSCONGLOMERATES c join sys.SYSTABLES t on c.TABLEID = t.TABLEID join sys.SYSSTATISTICS s on s.TABLEID = t.TABLEID<br> where t.TABLENAME = 'TEST_TAB_2' and c.ISINDEX = false;</p> 
                  <p>– There are still two statistic rows</p> 
                  <p>TABLEID CONGLOMERATENUMBER CONGLOMERATENAME ISINDEX ISCONSTRAINT CONGLOMERATEID TABLEID TABLENAME TABLETYPE STATID REFERENCEID TABLEID CREATIONTIMESTAMP TYPE VALID COLCOUNT STATISTICS<br> 84490209-0136-6999-c1b4-000065089f97 348432 84490209-0136-6999-c1b4-000065089f97 false false cccb420a-0136-6999-c1b4-000065089f97 84490209-0136-6999-c1b4-000065089f97 TEST_TAB_2 T edbc8255-0136-6999-c1b4-000065089f97 55410238-0136-6999-c1b4-000065089f97 84490209-0136-6999-c1b4-000065089f97 2012-03-31 17:36:49.629 I true 1 numunique= 1 numrows= 1<br> 84490209-0136-6999-c1b4-000065089f97 348432 84490209-0136-6999-c1b4-000065089f97 false false cccb420a-0136-6999-c1b4-000065089f97 84490209-0136-6999-c1b4-000065089f97 TEST_TAB_2 T 05278254-0136-6999-c1b4-000065089f97 63454207-0136-6999-c1b4-000065089f97 84490209-0136-6999-c1b4-000065089f97 2012-03-31 17:36:49.628 I true 1 numunique= 1 numrows= 1</p>

                  <p>insert into app.TEST_TAB_1 values (2);<br> insert into test_tab_2 values(2);</p> 
                  <p>– Update the statistics</p> 
                  <p>call syscs_util.syscs_update_statistics('APP', 'TEST_TAB_2', null);</p>
